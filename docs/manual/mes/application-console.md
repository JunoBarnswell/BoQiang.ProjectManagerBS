# MES 应用控制台和开发中心运维

## 这个功能做什么

应用控制台用于查看 MES 当前管理入口。开发中心用于维护菜单、页面草稿、预览和发布，是运维人员发布 MES 页面前最常用的区域。

## 应用控制台

![应用控制台](assets/screenshots/mes/application-console-annotated.png)

- 1：左侧菜单，进入应用控制台。
- 2：操作区，用于进入管理入口或刷新状态。
- 3：内容区，查看当前应用的管理卡片和入口。

### 常用操作

1. 登录 MES。
2. 点击“应用控制台”。
3. 查看开发中心、数据中心、审批流和系统设置入口。
4. 点击对应入口进入下一步操作。

### 遇到问题怎么办

- 控制台为空：刷新页面，确认当前账号是否有应用管理权限。
- 入口打不开：从左侧菜单直接进入同名功能；仍失败时记录截图。
- 进入了错误应用：退出后从正确的 MES 入口重新登录。

## 开发中心

![开发中心](assets/screenshots/mes/development-center-annotated.png)

- 1：左侧菜单，进入开发中心。
- 2：操作区，用于选择版本、刷新、新建页面、预览和发布。
- 3：内容区，查看菜单模块、页面草稿和页面状态。

### 常用操作

1. 进入“开发中心”。
2. 选择要维护的版本。
3. 在模块区域查看菜单结构。
4. 新建或打开页面草稿。
5. 修改页面后先保存。
6. 点击预览，确认页面显示和按钮效果。
7. 确认无误后执行发布。
8. 发布后在正式菜单中打开页面验证。

### 成功标志

- 页面草稿能保存。
- 预览页面能打开，内容与配置一致。
- 发布后正式菜单能看到页面。
- 业务用户能按权限正常访问正式页面。

## 版本、菜单和页面草稿

### 用途

- 版本：用于区分当前维护的是哪一套页面。
- 菜单模块：决定页面在左侧菜单中的位置。
- 页面草稿：保存还没有发布或正在修改的页面。

### 遇到问题怎么办

- 找不到版本：确认是否选择了正确应用，必要时联系管理员创建或启用版本。
- 菜单保存失败：检查菜单名称是否重复，必填项是否填写完整。
- 页面草稿找不到：确认是否在正确版本下查找。
- 预览不是最新：先保存页面，再刷新预览菜单后重新打开预览。
- 发布失败：查看页面提示，按提示补齐必填项、按钮动作、数据来源或授权。
- 发布后正式菜单没有出现：刷新菜单或重新登录；仍没有时确认是否发布到了正确版本和菜单位置。

## 正式页面和预览页面

### 区别

- 预览页面：给运维人员发布前检查，不建议业务用户日常使用。
- 正式页面：发布成功后出现在正式菜单，给业务用户日常使用。

### 遇到问题怎么办

- 预览能打开但正式页面没有：先确认是否已经发布成功。
- 正式页面仍是旧内容：确认用户是否刷新菜单，是否进入了正确页面。
- 两个页面内容不一致：以正式页面为业务使用准，回到开发中心检查草稿是否已发布。
