# 低代码页面发布运维手册

## 这个功能做什么

低代码设计器用于让运维人员在 MES 中创建和发布业务页面。它不要求运维人员写代码，重点是按步骤完成菜单、页面、字段、按钮、预览、发布、授权和审批绑定。

## 适用人员

- MES 运维人员：维护页面和菜单。
- 业务管理员：确认页面内容和字段是否符合业务。
- 审批管理员：确认页面是否需要绑定审批流程。

## 入口截图

- 1：左侧菜单，从 MES 进入开发中心和页面设计入口。
- 2：操作区，用于选择版本、刷新、新建页面、预览和发布。
- 3：内容区，查看模块、页面草稿和页面状态。

## 从 0 发布一个 MES 页面

### 第一步：进入开发中心

1. 登录 MES。
2. 点击“开发中心”。
3. 确认页面能看到版本、模块和页面草稿。

遇到问题怎么办：

- 找不到开发中心：联系管理员补充 MES 运维角色。
- 页面为空：刷新后仍为空，确认是否进入了正确应用。

### 第二步：选择版本

1. 在顶部或页面内选择要维护的版本。
2. 确认当前版本是本次要发布的版本。
3. 不确定时先不要发布，联系负责人确认。

遇到问题怎么办：

- 找不到版本：确认是否选错应用，或让管理员创建版本。
- 选错版本：不要继续修改，重新选择正确版本后再操作。

### 第三步：新建菜单模块

1. 在模块区域点击新增。
2. 填写模块名称，例如“生产报工”。
3. 选择放在左侧菜单中的位置。
4. 保存后确认左侧模块中出现新名称。

遇到问题怎么办：

- 保存失败：检查名称是否重复、必填项是否填写。
- 保存后看不到：刷新模块区域，确认是否选在了其他上级模块下。

### 第四步：新建页面草稿

1. 在目标模块下点击新建页面。
2. 填写页面名称。
3. 选择页面类型。
4. 保存后进入设计器。

遇到问题怎么办：

- 页面名称重复：换一个能区分业务含义的名称。
- 保存后未进入设计器：在页面草稿列表中重新打开该页面。

### 第五步：认识设计器区域

- 左侧组件区：选择页面要放什么，例如表单、表格、输入框、按钮、文本和图表。
- 中间画布区：摆放页面内容，确认用户最终看到的布局。
- 右侧属性面板：设置标题、字段、样式、数据来源和按钮行为。
- 顶部工具栏：保存、预览、发布、返回。
- 发布检查区：发布前查看是否有未完成项。
- 授权区：设置谁能看、谁能新增、编辑、删除、导入或导出。
- 审批绑定区：选择页面是否需要走审批。

遇到问题怎么办：

- 组件拖不上去：刷新页面后重新打开草稿。
- 属性面板不显示：先点击画布中的组件，再看右侧设置。
- 页面卡住：先保存截图，不要连续点击发布。

### 第六步：配置页面内容

常见组件用途：

- 容器：把页面内容分区。
- 表格：显示多条数据。
- 表单：新增或编辑一条数据。
- 输入框：填写文字、数字、日期等内容。
- 按钮：执行保存、删除、导入、导出、打开弹窗或跳转。
- 选择器：从选项中选择一个值。
- 图表：展示统计结果。
- 文本：显示标题、说明和提示。
- 媒体：展示图片或文件。

遇到问题怎么办：

- 字段不显示：确认字段已经添加到表单或表格。
- 下拉没有选项：检查字典或选项是否已维护。
- 按钮没反应：检查按钮是否设置了动作。

### 第七步：设置数据和查询

1. 确认页面要展示或保存哪些数据。
2. 给表格配置查询条件和显示列。
3. 给表单配置要填写的字段。
4. 给保存按钮配置保存动作。
5. 保存后进入预览页面检查。

遇到问题怎么办：

- 查询不到数据：先清空筛选条件，再确认是否有业务数据。
- 保存失败：检查必填项、重复值和页面提示。
- 表格列不对：回到设计器检查列设置。

### 第八步：设置授权和审批

1. 设置菜单可见范围。
2. 设置新增、编辑、删除、导入、导出等按钮的使用范围。
3. 如果页面需要审批，选择对应流程。
4. 保存后让测试账号登录确认。

遇到问题怎么办：

- 用户看不到页面：检查菜单授权和用户角色。
- 用户看得到但不能点按钮：检查按钮授权。
- 审批发起失败：确认流程已启用，页面已绑定流程，必填项已填写。

### 第九步：预览和发布

1. 点击保存。
2. 刷新预览菜单。
3. 从预览入口打开页面。
4. 按业务流程测试查询、填写、保存和按钮动作。
5. 通过发布检查后再发布。
6. 发布成功后从正式菜单打开页面复核。

遇到问题怎么办：

- 预览没更新：确认已保存，再刷新预览菜单。
- 发布被拦截：按提示逐项补齐，不要跳过检查。
- 发布后正式菜单没有出现：刷新菜单或重新登录。
- 正式页面异常：记录页面名称、截图、操作步骤和发布时间，升级给系统支持人员。

## 发布前检查清单

- 页面名称清楚，业务人员能看懂。
- 菜单位置正确。
- 必填字段已经设置。
- 表格列和查询条件正确。
- 按钮动作已配置并通过预览。
- 用户和角色已经授权。
- 需要审批的页面已绑定流程。
- 预览页面已由业务负责人确认。
- 发布影响范围已通知相关人员。

## 回显检查

发布后还需要重新打开草稿检查：

1. 页面布局是否还在。
2. 字段和按钮是否还在。
3. 授权和审批绑定是否还在。
4. 刷新浏览器后页面是否仍能打开。

如果回显不正确，先不要再次发布，记录截图并交给系统支持人员。
