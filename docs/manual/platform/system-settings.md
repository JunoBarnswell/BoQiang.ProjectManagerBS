# 平台系统设置运维

## 这个功能做什么

系统设置用于维护账号、组织、角色、菜单、字典、参数、公告、日志、在线用户、任务、文件和打印。它是平台日常运维最常用的区域。

## 用户管理

![用户管理](assets/screenshots/platform/system-users-annotated.png)

- 1：左侧菜单，从系统设置进入用户管理。
- 2：操作区，用于查询、新增、刷新、启用、停用或重置用户。
- 3：用户列表，查看账号、姓名、状态和行内操作。

### 常用操作

1. 进入“用户管理”。
2. 按账号、姓名或状态查询。
3. 新增用户时填写账号、姓名、手机号、所属部门和状态。
4. 用户离职或暂停使用时，先停用账号。
5. 保存后返回列表确认状态。

### 遇到问题怎么办

- 用户无法登录：先查账号是否存在、是否启用、是否被分配工作区和角色。
- 新增保存失败：检查账号是否重复、必填项是否为空。
- 用户看不到菜单：不要只改用户信息，要检查角色授权。
- 误停用：重新启用账号，并让用户重新登录。

## 角色管理

![角色管理](assets/screenshots/platform/system-roles-annotated.png)

- 1：左侧菜单，从系统设置进入角色管理。
- 2：操作区，用于查询、新增、保存角色和维护授权。
- 3：角色列表，查看角色名称、状态和操作入口。

### 常用操作

1. 进入“角色管理”。
2. 查询目标角色。
3. 新增角色时填写角色名称和说明。
4. 给角色勾选需要的菜单和按钮。
5. 保存后让相关用户重新登录。

### 遇到问题怎么办

- 授权后用户仍看不到菜单：确认用户确实拥有该角色，并让用户重新登录。
- 角色太多不好判断：优先按岗位或职责命名，例如“MES 运维员”“审批管理员”。
- 删除角色前提醒：先确认没有用户正在使用该角色，避免用户突然失去功能。

## 组织、岗位和菜单

### 用途

- 部门管理：维护公司、工厂、车间、班组等组织层级。
- 岗位管理：维护岗位名称，便于给用户归类。
- 菜单管理：检查菜单是否存在、是否显示、显示顺序是否正确。

### 遇到问题怎么办

- 部门选不到：确认部门已启用，并检查上级部门是否正确。
- 岗位显示异常：检查岗位名称是否重复或被停用。
- 菜单看不到：先查菜单是否启用，再查角色是否授权。
- 菜单顺序不对：调整排序后让用户刷新页面或重新登录。

## 字典、参数和公告

### 用途

- 字典管理：维护下拉选项，例如状态、类型、类别。
- 系统参数：维护系统开关和默认值。
- 通知公告：发布系统通知、停机通知或操作提醒。

### 遇到问题怎么办

- 下拉选项不显示：检查字典项是否启用，名称是否填错。
- 参数改完不生效：先确认保存成功，再让相关用户刷新页面；重要参数修改前要记录原值。
- 公告不显示：检查公告时间范围、状态和发布对象。

## 日志和在线用户

### 用途

- 操作日志：查询谁在什么时候做过什么操作。
- 登录日志：查询用户登录是否成功、登录时间和登录设备信息。
- 在线用户：查看当前在线人员，必要时让账号下线。

### 遇到问题怎么办

- 用户说自己没操作过：先查操作日志，按账号和时间过滤。
- 用户说登录不了：查登录日志，确认失败原因。
- 账号疑似被他人使用：查看在线用户，必要时强制下线并重置密码。

## 任务、文件和打印

### 用途

- 任务调度：查看定时任务是否正常执行。
- 文件中心：查看上传文件、下载文件和文件状态。
- 打印中心：查看打印模板、打印任务和打印结果。

### 遇到问题怎么办

- 任务没有执行：检查任务是否启用、最近执行时间和执行结果。
- 文件上传失败：检查文件大小、格式和网络状态。
- 文件下载失败：确认文件仍存在，当前账号是否有查看权限。
- 打印失败：确认模板存在、打印机可用、打印任务没有异常提示。
