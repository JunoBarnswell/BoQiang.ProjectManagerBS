# 平台管理运维

## 这个功能做什么

平台管理用于维护租户、应用、租户安装关系和用户在不同租户应用中的角色。它解决的是“谁能使用哪个应用、应用是否可进入、应用是否可发布”的问题。

## 谁会用

- 平台管理员：创建租户、维护应用、安装应用。
- 应用运维人员：进入应用后台、查看发布结果。
- 账号管理员：给用户分配租户和应用角色。

## 租户管理

![租户管理](assets/screenshots/platform/platform-tenants-annotated.png)

- 1：左侧菜单，从平台管理进入租户管理。
- 2：操作区，用于查询、新增、刷新、启用或停用租户。
- 3：租户列表，查看租户名称、状态和行内操作。

### 常用操作

1. 进入“平台管理”。
2. 点击“租户管理”。
3. 在查询区输入租户名称或关键字。
4. 点击查询，确认目标租户是否存在。
5. 新增租户时点击新增，填写名称、编码、状态等必填信息。
6. 保存后回到列表，确认新租户已出现且状态正确。

### 成功标志

- 列表能看到新增或编辑后的租户。
- 状态显示为启用时，后续才能为该租户安装和使用应用。

### 遇到问题怎么办

- 保存失败：先检查必填项和名称是否重复。
- 看不到新增按钮：当前账号没有新增权限，联系管理员授权。
- 停用后有人无法登录：先确认是否误停用租户，必要时重新启用并通知用户重新登录。
- 删除前提醒：删除会影响租户下的应用和用户使用，不建议在生产环境直接删除。

## 应用中心

![应用中心](assets/screenshots/platform/platform-applications-annotated.png)

- 1：左侧菜单，从平台管理进入应用中心。
- 2：操作区，用于查询应用、进入后台、发布、查看发布记录。
- 3：应用列表，查看应用名称、状态和可执行操作。

### 常用操作

1. 进入“应用中心”。
2. 搜索目标应用，例如 MES。
3. 查看应用状态是否正常。
4. 点击进入后台，切换到该应用的管理界面。
5. 如需发布，先确认当前版本和影响范围，再点击发布相关按钮。
6. 发布后查看发布记录，确认是否成功。

### 成功标志

- 点击进入后台后，页面切换到目标应用。
- 发布成功后，发布记录中能看到成功状态。
- 业务用户刷新后能看到新的正式页面或配置结果。

### 遇到问题怎么办

- 进入后台失败：确认应用是否启用，当前用户是否被分配到对应租户和应用角色。
- 发布失败：先查看页面提示和发布记录，确认是否有未填写项、未保存项或未通过检查项。
- 发布后用户看不到变化：让用户重新登录或刷新菜单；仍看不到时检查是否发布到了正确应用。
- 下载或删除发布结果失败：先确认账号权限，再确认该发布记录是否仍存在。

## 租户应用

### 用途

租户应用用于控制某个租户能不能使用某个应用。安装和启用后，用户才可能进入对应应用。

### 操作步骤

1. 进入“租户应用”。
2. 选择租户和应用。
3. 点击安装或启用。
4. 保存后回到列表确认状态。

### 遇到问题怎么办

- 安装失败：检查租户和应用是否都处于启用状态。
- 启用后用户仍看不到应用：继续检查用户租户关系和用户应用角色。
- 卸载前提醒：卸载会影响该租户下所有用户访问应用，必须先确认无业务使用。

## 用户租户关系

### 用途

用户租户关系用于决定用户属于哪些租户。没有租户关系时，即使有账号也进不了对应租户。

### 操作步骤

1. 进入“用户租户关系”。
2. 查询目标用户。
3. 给用户选择对应租户。
4. 保存后让用户重新登录确认。

### 遇到问题怎么办

- 用户看不到租户：确认关系已保存，并让用户重新登录。
- 选不到用户：先确认用户已创建且未停用。
- 选不到租户：先确认租户已创建且启用。

## 用户应用角色

### 用途

用户应用角色用于决定用户在某个租户、某个应用里能看到哪些菜单、能点哪些按钮。

### 操作步骤

1. 进入“用户应用角色”。
2. 选择用户、租户和应用。
3. 分配合适角色。
4. 保存后让用户重新登录。

### 遇到问题怎么办

- 用户能进应用但菜单少：检查应用角色是否正确。
- 用户有菜单但按钮不可点：检查角色是否包含对应操作。
- 授权后不生效：让用户退出后重新登录；仍不生效时检查是否分配到了正确租户和应用。
