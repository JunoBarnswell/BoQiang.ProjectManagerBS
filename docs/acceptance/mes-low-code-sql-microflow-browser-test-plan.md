# MES 低代码 SQL 微流运行页人工验收计划

## 验收边界

本计划只允许使用 Codex 内置浏览器进行人工操作：打开页面、点击可见按钮、输入可见表单、刷新页面、读取可见列表和错误提示。不得使用脚本验收、浏览器控制台、接口请求、数据库查询、自动化测试代码或通过修改数据库结果代替页面证据。

工程构建、单元测试和数据库 fixture 初始化仅作为开发门禁，不计入人工验收通过证据。

## 前置条件

- 后端监听 `http://127.0.0.1:5000`，前端监听 `http://127.0.0.1:5173`。
- 使用具备 MES 应用开发中心设计、发布和运行页 `view/add/edit/delete` 权限的已登录账号。
- 固定设计页：`/tenants/tenant-a/apps/MES/admin/development-center/pages/1d77fd8dee5941aeba46fd19a650c360/designer`。
- 发布后运行页：`/tenants/tenant-a/apps/MES/admin/pages/page_mr7xi5jk`。
- 页面必须通过设计器可见的“保存/发布”动作生成正式运行页，不能直接调用发布接口。

## 用例

| 编号 | 人工操作 | 可见通过标准 |
| --- | --- | --- |
| B-01 | 打开设计器，确认页面名称、ERP 风格顶部工具栏、查询区、订单表格和编辑区 | 设计器加载成功，无错误横幅；页面包含“MES订单管理”、关键字/状态/日期查询、订单表格和新增/修改操作 |
| B-02 | 在设计器点击保存，再点击发布；进入发布后的运行页 | 设计器显示发布成功；运行页地址为 `page_mr7xi5jk`，刷新后仍能打开；菜单/页面标题与 MES 订单一致 |
| B-03 | 在运行页首次加载，观察订单列表；输入关键字或状态后点击查询 | 列表由 SQL 微流返回；查询条件变化不会出现未处理异常；查询后结果与条件可见一致 |
| B-04 | 点击新增，填写唯一订单号 `BROWSER-MES-CRUD-001`、客户、订单日期、金额和状态，点击“新增保存” | 页面给出成功反馈；点击查询后新订单出现在表格中；刷新页面后记录仍存在 |
| B-05 | 在该行点击“载入编辑”，修改客户或备注，点击“编辑保存” | 页面给出成功反馈；重新查询可见修改后的值；刷新页面后修改仍存在 |
| B-06 | 在该行点击“删除” | 页面先显示确认区域；点击“取消”时记录保留；再次点击“删除”并确认后给出成功反馈，查询结果中记录消失；刷新后仍不存在 |
| B-07 | 对存在明细的订单执行删除 | SQL 微流拒绝删除并显示业务错误；订单仍保留，不得出现半删除或页面假成功 |
| B-08 | 在发布运行页刷新、从应用菜单重新进入，再返回设计器 | 页面状态和数据一致；正式运行页不依赖设计器草稿；设计器仍可查看当前发布版本 |
| B-09 | 使用无 `edit` 或无 `delete` 权限的账号尝试打开/操作（若环境提供该账号） | 页面动作隐藏或不可用；即使尝试操作，后端显示统一无权限错误；不能以匿名 `401` 作为通过证据 |

## 证据与结论

每个用例只记录内置浏览器中的可见结果和截图/页面状态。结论分为：

- `Pass`：页面操作和刷新后的可见结果都符合标准。
- `Fail`：页面可操作但结果、权限或持久化不符合标准。
- `Blocked`：浏览器无法连接、没有有效登录身份或无法提供相应权限账号；不得以构建或接口结果替代。

最终发布结论要求 B-01 至 B-08 全部 `Pass`；B-09 在没有可用低权限账号时只能标记 `Blocked`，不能伪造通过。
