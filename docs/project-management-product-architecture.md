# ProjectManagement 产品与领域架构

## 1. 产品主线

ProjectManagement 不是项目列表加若干页面，而是一个以 `Project` 为工作边界、以 `Task` 为唯一执行对象的计划系统：

```text
工作区
  -> Project
      -> Member / Role / Scope
      -> Milestone
      -> Task Tree
          -> Participant / Label / Dependency / Time Log / Comment / Attachment
      -> View / Activity / Notification / Report
```

所有视图只改变投影方式，不改变任务命令、权限、数据过滤、版本和错误语义。

## 2. 聚合与责任

### Project 聚合

Project 保存编码、名称、生命周期状态、负责人、日期、WIP 和项目级进度。创建时在同一事务中创建唯一 Owner 成员；删除、恢复和归档不允许绕过版本号。项目可见性由工作区过滤器与 Owner/Member 关系共同决定。

### Membership 聚合

成员角色固定为 `Owner / Manager / Lead / Member / Viewer`。角色是项目级授权边界，不是页面显示字段。至少保留一个有效 Owner；Lead 的可管理范围由 `ScopeRootTaskId` 表达，并且后端命令必须重新校验后代关系。

### Milestone 投影

Milestone 是独立实体，不用任务布尔字段替代。任务最多关联一个里程碑。里程碑进度由叶子任务的 `ProgressPercent * Weight` 加权汇总；健康状态由生命周期、截止日期和进度统一计算为 `OnTrack / AtRisk / OffTrack / Done`。

### Task 聚合

Task 的父子树、状态、进度、日期、WIP、负责人、依赖和版本构成唯一命令边界。父任务移动必须同步整棵子树深度；禁止跨项目移动、循环和超过 10 层。任务状态使用唯一状态机：

```text
Todo -> InProgress -> Blocked -> InProgress -> Done
  \       \            \-> Cancelled
   \       \-> Done
    \-> Cancelled
```

依赖产生的阻塞由系统计算，不允许前端直接伪造解除；强制开始必须有角色权限、原因和审计。

## 3. 统一命令与查询

命令入口按聚合划分：

- Project：创建、编辑、状态变更、归档、删除、恢复。
- Membership：添加、角色/范围变更、移除。
- Milestone：创建、编辑、状态变更、删除。
- Task：创建、编辑、移动、批量更新、删除、恢复。
- Dependency：添加、删除、阻塞刷新。

所有写命令都带 `VersionNo`，返回新的版本；失败统一返回业务错误和 TraceId。所有查询从当前 session 的 `TenantId + AppCode` 开始，ORM Data Filter 负责租户/应用/项目可见性，查询参数只能表达排序、筛选和投影。

任务查询协议唯一使用 `ProjectManagementTaskQuery`，`ViewKey` 只允许 `tree/list/card/board/gantt/calendar`。五种视图共享同一筛选、分组、排序、分页、阻塞状态和 QueryKey，禁止为视图复制接口。

## 4. 权限模型

Controller 权限码负责菜单/动作的第一层拦截；应用服务必须校验项目角色和任务范围。Viewer 只读，Member 可编辑被分配任务，Lead 只能编辑授权子树，Manager 可管理项目范围，Owner 可执行高风险操作。跨租户、跨应用、跨项目和篡改请求体中的工作区字段均拒绝。

## 5. 失败与一致性

关键写操作遵循：

```text
校验工作区与权限
 -> 校验聚合关系和版本
 -> 校验循环/WIP/依赖/引用
 -> 同一数据库事务写入
 -> 生成活动/通知/缓存失效事件
 -> 返回新版本与影响摘要
```

重复提交使用业务唯一键或版本冲突拒绝；删除有引用对象返回明确业务错误；批量操作使用全量预检，失败整体回滚，不产生部分成功的假象。

## 6. UI 信息架构

- 项目中心：项目列表、筛选、收藏、最近项目。
- 项目工作区：概览、任务、看板、甘特、日历、里程碑、报表、设置。
- 我的工作：今日、未来、逾期、阻塞、提及和分配任务。
- 协作：评论、Mention、附件、活动、通知。
- 数据中心：搜索、导入导出、同步、备份恢复、回收站、审计。

页面必须显式覆盖 Loading、Empty、Error、403、404、冲突、未保存和后台任务状态；深链刷新不得丢失工作区与项目上下文。

### 项目关联 IM 边界

项目管理只维护 `Project/Task -> IM Conversation` 的关联、来源成员快照和同步策略；IM 模块继续唯一负责会话、消息、未读、投递和历史。关联以 `TenantId + AppCode + ProjectId + TaskId?` 为幂等业务键：项目可绑定一个群会话，重点任务可绑定一个独立会话。创建、重试和并发请求必须返回同一关联，禁止复制消息到任务评论。

项目 Owner/Manager 发起关联时，服务端先校验项目角色，再以项目成员或任务参与人为准生成目标参与人；成员增加、移除、项目归档或任务删除通过项目域事件驱动同步。移除成员必须先撤回其 IM 会话访问，再提交项目成员变更后的实时失效；会话历史保留但标记关联对象已归档/删除，禁止失去项目权限的用户通过旧会话读取消息。

`pm_im_conversation_links` 是项目域的持久化编排记录，使用确定性 `ConversationKey` 与项目/任务作用域唯一键处理重试和并发。项目应用库与 IM 主库不共享数据库事务：链接先进入 `Provisioning`，IM 会话成功后才切为 `Active`；失败保留错误用于可观测重试，绝不伪造跨库原子提交。项目删除归档全部关联会话；项目恢复仅恢复项目会话，任务会话只能随对应任务（或任务子树）恢复而重启。

任务会话成员是当前有效项目成员的受控子集：Owner/Manager/Lead、任务负责人和显式任务参与人；任何参与人都必须仍是有效项目成员。项目成员移除优先从项目和任务会话移除该用户，任务参与人移除只影响对应任务会话；任务负责人、项目成员或任务参与人变更后重新按这一唯一规则收敛成员。

从任务跳转 IM 与从消息跳回任务都必须携带受服务端签发的关联标识，落地时重新校验当前 TenantId、AppCode、项目成员关系和任务可见性。前端不得根据 URL、缓存成员或 IM 参与人自行决定项目访问权限。

## 7. 数据中心与同步一致性

数据中心不直接把当前数据库快照当作同步协议。所有任务写命令在业务写入成功后追加 `pm_sync_journal`，日志包含 `TenantId + AppCode + ProjectId`、单调 `SequenceNo`、聚合版本、操作类型、完整变更载荷、操作者、设备和 TraceId。ORM 过滤器同时限制工作区与项目成员范围，不能通过修改请求体读取其他租户、应用或项目的日志。

`.bqsync` 包固定包含 `manifest.json` 与 `data.json`，附件使用独立条目并通过 SHA-256 校验。导入必须遵循：

```text
权限与工作区预检
 -> ZIP 结构、大小、路径、版本和校验和验证
 -> 项目/任务冲突预览
 -> 当前密码确认与风险确认
 -> 维护锁、自动备份
 -> 按依赖顺序事务导入
 -> 健康检查、附件补偿清理
 -> 提交；失败则回滚并保留失败历史
```

预览不得写库；`Skip / Overwrite / Reject` 冲突策略必须由服务端执行。设备水位只能来自服务端同步日志，不能由前端自报已完成序号。备份、恢复、回收站和审计中心共享同一高风险操作记录与权限边界。

高风险命令获取 `pm_maintenance_locks` 后才能进入执行阶段。锁按 `TenantId + AppCode + LockKey` 唯一，具备过期回收和 finally 释放语义；没有锁、锁已被占用或锁超时的请求不得继续写数据。

SQLite 数据空间备份使用在线备份 API 生成物理文件并记录 SHA-256；恢复前先生成安全备份，恢复后执行 `PRAGMA integrity_check`，校验失败自动回滚安全备份。备份文件路径只能落在服务端受控目录，数据库不是 SQLite 或是内存库时明确拒绝该能力。

高风险同步导入、备份和恢复统一写入 `pm_operations` 操作记录，状态按 `Running -> Succeeded/Failed` 流转，记录影响摘要、TraceId、操作者、开始/完成时间及失败原因；审计中心通过独立查询接口展示这些记录。操作记录写入失败不得覆盖原始业务异常，业务失败也必须尽力留下失败证据。

## 8. 交付准入

Case 只有在代码、自动化测试、真实接口/页面、权限允许/拒绝、租户应用隔离、边界失败和 Linear 证据全部存在后进入 `Done`。任何主题父任务必须等待所有子 Case 完成并执行一次主题级全量回归。
