# 项目管理响应式、键盘与可访问性交互规范

> Linear：`HAO-436`（M0）
>
> 目标：为项目中心、项目工作区（任务树/列表、卡片、看板、甘特、日历）和项目治理页面定义可验证的响应式、键盘、辅助技术、触摸/鼠标与跨浏览器行为。本文区分当前实现基线与待交付要求；**待交付要求不得被视为当前可用功能**。
>
> 事实基线：`docs/project-management-information-architecture.md`、`docs/project-management-user-scenarios-glossary.md`、`frontend/AsterERP.Web/src/core/responsive/breakpoint.ts`、`ResponsiveModal.tsx`、`DataTable*.tsx`、`TaskWorkspaceToolbar.tsx`、`TaskWorkspaceProjection.tsx`、`TaskWorkspaceSelectionPanel.tsx`。

## 1. 适用范围、状态标识与共同原则

| 标识 | 含义 | 本文中的使用方式 |
|---|---|---|
| **当前基线** | 源码已存在的行为；仍须在真实浏览器复验。 | 作为设计约束和回归测试起点，不能以静态阅读替代验收。 |
| **目标要求** | HAO-436 定义、但源码尚未完整满足的可访问性交互。 | 后续实现必须按本规范补齐并做浏览器验收。 |
| **追溯 Case** | 该能力由明确的后续 Linear Case 承接。 | 未完成前，UI 不得暗示该能力可用。 |

1. 项目工作区以当前认证会话的 `TenantId + AppCode` 和路由 `:projectId` 为边界；键盘、拖拽、浏览器缓存或深链均不得绕过服务端权限与对象数据过滤。
2. 任务树/列表、卡片、看板、甘特和日历是同一 Task Workspace 的投影，切换投影必须保留其合法 URL 查询、选中任务和权限语义；不可因视图不同产生第二份不可解释的焦点、草稿或排序状态。
3. 鼠标、触摸和键盘是并列输入方式。任何会改变任务父子关系、状态、排序、日期或持久化数据的鼠标动作，必须有同等语义的键盘或明确命令入口；失败时保持可恢复状态。
4. 权限隐藏是体验层。即使控件不可见或禁用，直接 API 调用仍由服务端 `PermissionCode`、对象权限和 ORM Data Filter 拒绝；拒绝页/错误页不能泄露对象存在性。

## 2. 响应式基线与验收视口

### 2.1 当前断点事实

`ERP_BREAKPOINTS` 当前定义为 `xs=0`、`sm=768`、`md=1024`、`lg=1366`、`xl=1600`、`xxl=1920`、`ultra=2560`；`<1366` 为 `compact` 密度。响应式弹窗在 `<768` 为全屏、`768–1023` 为抽屉、`>=1024` 为模态框，标准工具栏在 `1024–1599` 默认最多保留三个动作，其余进入原生 `<details>` 溢出菜单。

这只说明共享组件的当前阈值。Task Workspace 的看板、甘特和日历未因上述通用阈值自动获得可用性保证，必须以本节的真实视口用例验收。

| 验收档位 | 视口与缩放 | 必须验证的页面 | 可用结果 |
|---|---|---|---|
| 1024 临界 | `1024×768`，浏览器 100% | 项目中心、任务六种投影、详情/编辑、治理页 | 主操作可达；表格/甘特可在自身滚动容器横向滚动，但页面根部不产生意外横向滚动；不会因工具栏折叠丢失保存、取消、筛选、视图切换、创建任务或返回操作。 |
| 平板横屏 | `1180×820`，触摸与鼠标各一次 | 卡片、看板、甘特、日历、附件上传、日期输入和弹窗 | 控件可点击/触达；不以 hover 才能发现关键信息；抽屉/模态框不被浏览器键盘或安全区遮挡。 |
| 桌面 | `1440×900`，浏览器 100% | 全部主路径及批量操作 | 使用 `lg` 以上标准密度；多列信息可读，列固定、滚动和焦点不遮挡。 |

**目标要求：** 验收另应在 200% 浏览器缩放（等效窄宽）重复完成任务创建、查看详情、筛选和保存；除数据密集型表格/甘特的明确内部滚动容器外，禁止双向页面滚动。`1024×768` 不是仅截图检查，必须完成一条真实的项目—任务—保存/失败恢复路径。

| 区域 | 当前基线 | 1024 / 平板横屏目标要求 | 追溯 |
|---|---|---|---|

## 3. 焦点、键盘与快捷键

### 3.1 全局焦点规则

1. **目标要求：** 所有可交互元素必须使用原生控件，或具有正确 `role`、可访问名称、`tabIndex=0` 与 Enter/Space 行为；不得把 `div`/`span` 的 `onClick` 或拖放事件当成可键盘操作控件。当前任务根投放区和“作为子任务”投放区是 `div`/`span`，仅有 HTML5 拖放事件，尚不满足此要求。
2. Tab 依 DOM 和视觉阅读顺序在工作区标题、视图导航、筛选、工具栏、内容、详情/抽屉中前进；Shift+Tab 反向。不得以正数 `tabIndex` 重新排序。
3. 焦点必须始终可见，并在固定表头、粘性列、抽屉和滚动容器中自动滚入可视区域；高对比/焦点样式不能仅通过颜色差异表达。
4. 页面应提供“跳到主内容”链接；项目/任务标题区、主内容、任务详情应具有稳定的 landmark 或可命名区域。此跳转链接目前未在本模块基线中确认，属于目标要求。
5. 触发异步刷新、筛选、分页或切换视图时，保持触发控件焦点；若主动打开详情/错误摘要，焦点移到该区域的标题或首个可修复字段，且不静默跳到页面顶部。

### 3.2 弹窗、抽屉与确认

| 规则 | 当前基线 | 目标要求 / 验收 |
|---|---|---|
| 打开 | `ResponsiveModal` 有 `role="dialog"`、`aria-modal="true"` 与标题文本 `aria-label`，且监听 `Esc`。 | 记录触发元素；打开后将焦点放在标题、首个错误字段或首个可操作控件。 |
| 焦点圈定 | 当前未见 focus trap 或关闭后的焦点恢复。 | Tab/Shift+Tab 只能在最顶层模态框内循环；关闭后回到原触发元素，若元素已被移除则回到其逻辑父级。 |
| 关闭 | 当前模态框 `Esc` 与遮罩点击可关闭；`ResponsiveModal` 的关闭按钮仅显示 `×`，未见可读名称。`ModalForm` 的关闭按钮已有 `aria-label`。 | `Esc` 只关闭最顶层、允许关闭的层；有未保存修改时先进入确认，不丢草稿。所有关闭图标必须有本地化可访问名称和 tooltip。 |
| 标注 | 当前以 `aria-label={title}` 标注。 | 使用唯一标题 ID 的 `aria-labelledby`；有说明/风险/错误摘要时用 `aria-describedby` 关联。 |

### 3.3 快捷键与冲突规约

| 快捷键/按键 | 范围 | 规范 | 当前状态与追溯 |
|---|---|---|---|
| `Tab` / `Shift+Tab` | 全页面 | 遵循第 3.1 节焦点顺序；不劫持为切换任务或拖拽。 | 原生控件可用；完整焦点路径待 `HAO-528` 浏览器验收。 |
| `Enter` / `Space` | 原生按钮、链接、checkbox、选择框 | 执行其可见操作；不得通过空白卡片捕获来误触保存或移动任务。 | 现有任务标题/工具栏多数为原生控件。 |
| `Esc` | 最顶层弹窗、抽屉、拖拽模式 | 依次取消拖拽、关闭临时菜单/筛选，再关闭可安全关闭的弹窗；有脏数据时先确认。 | 仅 `ResponsiveModal` 当前监听 `Esc`；脏草稿/优先级处理待 `HAO-466`、`HAO-478`、`HAO-521`。 |
| `Ctrl/Cmd+K` | 非编辑字段中 | 仅在全局搜索上线后打开搜索并将焦点放入搜索框；输入、textarea、可编辑区内不得劫持。 | 当前不可宣称已支持；`HAO-503` 交付后方可启用。 |
| 单字母快捷键 | 全页面 | 默认不注册，避免与输入法、屏幕阅读器和浏览器快捷键冲突；若后续新增，必须可发现、可关闭且不在文本输入中生效。 | 目标约束。 |

### 3.4 键盘拖拽与替代命令

当前 `TaskWorkspaceProjection` 与共享 `useDragReorder` 仅使用 HTML5 `draggable`、`dragstart/dragover/drop`；它们不提供键盘或触摸拖拽。因此当前只能将鼠标拖放视为待浏览器验证的基线，不能宣称任务移动已无障碍。

**目标要求（HAO-472 为主，事务/排序由 HAO-444，日期视图由 HAO-475/HAO-477）：**

1. 每项可移动任务提供有名称的“移动任务”原生按钮/拖拽句柄，并有不依赖拖拽的“移动到…”命令菜单。仅有 `draggable` 的卡片、行、根投放区和子任务投放区不得作为唯一入口。
2. 焦点在移动按钮时，Space 进入“已拾取”模式，并以 `aria-live="polite"` 公布任务名称、当前位置及可用按键；再次 Space 或 Enter 提交，Esc 放弃且不写入。
3. 树/列表中：ArrowUp/ArrowDown 在同级前后移动；ArrowRight 缩进为前一个同级任务的子任务（不合法时公布原因）；ArrowLeft 提升到父级之后。看板中：ArrowLeft/ArrowRight 选择相邻状态列，ArrowUp/ArrowDown 选择列内位置。所有候选变更先通过当前项目、层级循环、权限、WIP、依赖和 `VersionNo` 校验，再提交。
4. 甘特/日历不以 Arrow 直接静默改日期：键盘先进入“编辑日期”命令，输入/选择开始与截止日期，再进行相同的服务端边界和并发验证。
5. 成功后将焦点放在移动后的任务，并公布新父任务/状态/位置；失败、冲突或权限拒绝时保留原焦点与本地选择，展示可读原因和重试/刷新/取消路径。

## 4. 语义、名称、状态与辅助技术反馈

| 范畴 | 当前基线 | 目标要求 |
|---|---|---|
| 导航与表单 | 任务视图 nav 已有可读 `aria-label`；任务编辑字段、评论、附件和文件输入多有 `aria-label`；`AdaptiveSearchForm` 是原生 `form`。 | 所有表单字段优先使用可见 `<label>`；仅在视觉无法显示时才用 `aria-label`。必填、格式、范围、错误说明与字段用 `aria-describedby` 关联。 |
| 表格 | `DataTable` 使用原生 `table/thead/tbody/th/td`；排序/筛选为按钮，筛选按钮已有可访问名称。 | 排序按钮须以 `aria-sort` 或等价可读状态说明当前字段、方向和多列优先级；分页前/后按钮当前无名称，必须补“上一页/下一页”，并在禁用时保留原因可读性。 |
| 图标按钮 | `PermissionButton` 内部图标为 `aria-hidden`，文字子项可成为名称。 | 只显示图标的按钮必须给本地化 `aria-label`；`title` 不能作为唯一名称。`ResponsiveModal` 的 `×` 是当前已识别缺口。 |
| 任务卡片/拖放 | 标题按钮、checkbox 和 `article` 已提供部分语义。根/子投放区却是无焦点 `div/span`。 | 投放、移动、选择和展开必须具有可达控件、当前状态与指令；不可把 `aria-label` 放在不可聚焦 `div` 后视为完成。 |
| 状态/颜色 | 当前任务/项目状态通常有文字，如 `Todo`、`Blocked`、`Done`；部分错误框也有文字。 | 状态必须始终有文本或图标+文本，不只用颜色、边框、拖放高亮或动画；图例/筛选项使用词汇表中的统一状态。视觉令牌由 `HAO-433`，原型一致性由 `HAO-434`。 |
| 加载、成功与错误 | 任务保存/评论有“保存中…”/“发布中…”文本；`PageStateShell` 提供标题、描述和操作，但未见 live region。 | 提交区域设置 `aria-busy`；操作结果以合适的 `role="status"`（成功/进度）或 `role="alert"`（需立即处理的失败）公布一次，不重复朗读。错误不得只变色或只显示 toast。 |
| 权限与空状态 | `PermissionButton` 可按权限隐藏或禁用；`PermissionGuard` 可给 `Page403`。 | 隐藏动作不应留下无名称空白；禁用的已显示动作应说明缺少条件（权限、必填项、处理中、归档只读、依赖/WIP 等），且 API 拒绝信息不暴露无权对象。 |

## 5. 鼠标、触摸、加载、错误、冲突与未保存状态

### 5.1 指针输入

1. **目标要求：** 主要按钮、图标按钮、checkbox 标签、拖拽句柄和看板投放入口的可点击区域最小为 `44×44` CSS px，彼此紧邻时保留足够间距；当前 `DataTablePagination` 的 `28px` 图标按钮不满足此目标，须在共享组件中修正或提供等效大点击区。
2. 关键内容不能只在 `hover` 显示；触摸设备必须可通过点击/焦点看到相同的任务操作、说明与错误。
3. 不依赖 HTML5 drag-and-drop 在触摸浏览器中的实现差异。触摸可用长按/拖动作为增强，但“移动到…”命令始终存在，并沿用第 3.4 节的权限、并发和反馈规则。
4. 文件上传必须保留原生 file input 或等价可键盘触发的标签，显示文件名、大小、上传中、成功/失败及重试/移除；`HAO-487` 负责附件闭环。

### 5.2 状态机与恢复

| 状态 | 必须行为 | 当前事实 / 追溯 |
|---|---|---|
| 禁用 | 原生 `disabled` 防止重复提交；紧邻文本或描述说明原因；禁用不等同未授权。 | `PermissionButton` 已将 `disabled || !hasPermission` 下传；原因文字仍为目标要求。 |
| 加载/提交中 | 禁止同一命令重复提交，保留已输入内容；显示进行中名称、`aria-busy` 和非阻塞取消/返回规则（若业务允许）。 | 任务保存/评论已有文字和 disabled；区域忙碌语义待补。 |
| 业务/网络错误 | 不清空筛选、选择、任务草稿或文件选择；在可见位置说明失败动作、原因/TraceId（若可安全公开）和重试/返回。 | 现有页面有文本错误状态；统一恢复及浏览器验收由 `HAO-466`、`HAO-478`、`HAO-528`。 |
| 乐观更新失败 | 立即回滚临时视觉状态，聚焦回触发任务，公布“未保存”；不得把本地移动当成功。 | `HAO-472` 承接。 |
| `VersionNo` 冲突 | 保留本地草稿和服务端最新版本，明确字段/动作冲突；提供刷新、比较、重新应用或取消，不得静默覆盖。 | 用户场景词汇表已定义版本冲突；任务详情闭环由 `HAO-466`，批量/移动由 `HAO-444`。 |
| 未保存离开 | 内部导航、切换视图、切换项目/工作区、关闭抽屉和 `beforeunload` 均应拦截脏草稿；确认“保存、放弃、继续编辑”。 | 当前项目表单有 `beforeunload` 基线，Task Workspace 未完成统一保护；`HAO-478`、`HAO-521` 承接。 |
| 归档/无权限/删除 | 变为只读或清理已失效的 `taskId`、缓存和订阅；提供安全的返回路径，不展示可写成功假象。 | 路由/治理约束见信息架构；`HAO-415`、`HAO-438`、`HAO-454`、`HAO-478` 承接。 |

## 6. 跨浏览器测试矩阵

### 6.1 范围与执行原则

必测为当前稳定版 Windows Chrome、Edge、Firefox；补充验证为当前稳定版 macOS Safari 和 iPadOS Safari，重点覆盖触摸、日期选择器、文件输入、键盘与弹窗。使用验收当日可获得的稳定版本，不把固定版本号写入长期规范。当前前端脚本提供 `lint`、`test`、`typecheck`、`build`，但未见项目管理专用的浏览器端到端矩阵；因此以下结果在实际执行前均为**未验证**，不得报为 Pass。

## 7. 交付门槛与可追溯清单

HAO-436 的文档交付完成不代表产品可访问性已完成。后续实现/验收必须至少满足以下清单：

1. 在 `1024×768`、`1180×820`、`1440×900` 和 200% 缩放完成项目中心到任务详情的一条成功、一条失败/冲突路径。
2. 完成弹窗焦点进入、Tab 环绕、Esc、关闭恢复、脏草稿确认和无名称图标按钮检查。
3. 在树、列表、看板、甘特、日历逐一完成非鼠标任务移动；鼠标/触摸/键盘结果、服务端校验、失败回滚和公告一致。
4. 完成表格语义、分页名称、状态非颜色化、加载/错误 live region、禁用原因和最小触达区检查。
5. 按第 6 节执行跨浏览器矩阵，并将 Pass/Fail/Blocked 证据附到承接 Case；任何 Blocked 必须说明环境、风险及可复现步骤。
