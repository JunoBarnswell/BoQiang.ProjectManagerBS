# Low-Code Studio quality gates

These gates protect the current production chain: `RuntimePage.tsx` loads the published page schema, `RuntimePageController` serves it, and `ComponentRuntimeHost.tsx` verifies and renders the latest runtime artifact. Migration services, rejection validators, fixtures, and test files are inputs or evidence; they are not runtime entry points.

The CI workflow runs all deterministic gates and uploads `artifacts/` even when a gate fails. The external release boundary is defined by `docs/low-code-refactor/external-evidence-contract.json` and requires these exact records under `artifacts/low-code-external/`: `api-ui-auth.json`, `provider-sqlserver.json`, `provider-mysql.json`, `provider-postgresql.json`, `provider-sqlite.json`, `rollback.json`, `performance.json`, `visual-accessibility.json`, and `observability.json`. Each record must contain non-empty `evidenceId`, `status`, `scope`, `source`, `validatedAt`, and `traceId`; its `status` must be exactly `Pass` and it must not carry a non-empty `blockedReason`. A single arbitrary `status=Pass` file is not sufficient. Missing evidence, `Blocked`, `EvidencePresent`, or any other status fails the workflow; it is never converted to a CI pass. The repository currently has no complete set of such external evidence, so that gate remains an explicit release blocker until a controlled run supplies it.

## Deletion policy

The scan protects `backend/AsterERP.Api`, `backend/AsterERP.Contracts`, `frontend/AsterERP.Web/src`, and `docs/contracts`. It excludes compiled output, static assets, database files, fixtures, backend `*Tests.cs`, and frontend `*.test.ts/tsx`. Only explicitly named migration/rejection files may contain old migration vocabulary. This is a narrow input boundary, not a production allowlist.

Every guard must identify its production file/symbol and fail on a real regression. Do not silence a finding by widening an exclusion or adding a broad allowlist.

## C01-C15 evidence matrix

`Pass` means the named command completed with retained output. `EvidencePresent`
means the repository contains the named test/fixture and a local result is
recorded, but it is not a release approval. `Blocked` means a required browser,
provider, authorization, maintenance-window, or rollback artifact is missing.
Blocked and EvidencePresent must never be converted to Pass by a
documentation-only update. This rule is also enforced by the executable
quality-gate test.
